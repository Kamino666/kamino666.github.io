论文笔记 Video Captioning via Hierarchical Reinforcement Learning

本文最后更新于：2023年2月8日 下午

论文笔记 Video Captioning via Hierarchical Reinforcement Learning

论文链接：CVPR 2018 Open Access Repository (thecvf.com)

代码链接：无

文章通过提出一个新颖的**分层强化学习网络（Hierarchical Reinforcement Learning, HRL）**来提升Video Captioning任务的细节详细描写效果，HRL包含一个高级的Manager模块和低级的Worker模块，前者会设计出一些句子生成的子目标，而后者将会实现这些子目标。效果在现今看来并不算好，但思路比较有趣。

简介

Video Captioning可以分成单句生成和段落生成两个方向，后者会生成多条句子来描述，如下图所示，视频内容包含了sit on bed、put a laptop into a bag，然后接着stand up、put the bag on one shoulder、``walk out`这些动作，这种细粒度的描述对模型提出了更高的要求。

本文提出了一种分而治之的解决方案，其将长句子分为多个小的文本段（比如上图中不同颜色的部分），然后通过一个序列模型来生成每个文本段的具体内容。与其让模型直接生成整个长句子，这种方式可以引导模型一步步地生成。一个高级的序列模型会设计每个段落的上下文，另一个低级的序列模型则会跟随上下文的引导来生成每个段落的具体单词。

此外，本文还通过处理Charades数据集获得了一个新的Charades Captions数据集，这个数据集包含更细粒度的视频描述（上图例子就是之一）。

模型架构

如图所示，编码阶段，CNN提取的特征会依次经过一个Bi-LSTM提取低级特征（hEwh^{E_w}hEw​）和一个LSTM提取高级特征（hEmh^{E_m}hEm​），得到的特征被HRL Agent利用，而Agent会和Environment进行交互。这里Agent就相当于Video Captioning模型的解码器。

HRL Agent包含了一个Worker、一个Manager和一个Internal Critic。Manager在时序上精度更低，只是大概地生成任务（goal），Worker则根据goal一步步生成单词来完成它，Internal Critic则判断Worker是否完成了任务，并发送一个信号给Manager来决定是否生成下一个任务。

注意力模块

解码器（或叫HRL Agent）中的Worker和Manager都使用了注意力模块。

如下图，worker的上下文向量ctWc^{W}_tctW​是由编码器输入的特征加权相加得到，而权重又是通过LSTM的当前隐藏向量ht−1Wh^W_{t-1}ht−1W​和编码器的特征经过MLP得到的。manager的注意力模块也类似，只是输入的特征向量不同而已。

Manager和Worker

Manager的输入是注意力得到的编码器上下文向量ctMc^M_tctM​和Worker隐藏向量ht−1Wh^W_{t-1}ht−1W​的拼接（因为是LSTM，还包括自身传递的隐藏向量ht−1Mh^M_{t-1}ht−1M​），其产生的输出为htMh^M_thtM​，在经过一个uMu_MuM​映射之后就是goal向量gtg_tgt​。

Worker的输入是注意力得到的编码器上下文向量ctWc^W_tctW​、goal向量gtg_tgt​和上一步的单词输出at−1a_{t-1}at−1​（同样也有自身传递的ht−1Wh^W_{t-1}ht−1W​，aaa是action，也就是产生的单词），其产生的输出为htWh^W_thtW​，经过映射得到xtx_txt​，然后softmax得到概率πt\pi_tπt​。
