强化学习能干什么？

强化学习可以让计算机学会操控机器人、玩游戏等……（原文写的很简略，大家也都知道强化学习的各种应用了，就不多说了）

关键概念

强化学习最重要的就是Agent和Environment了，在每一步交互中，Agent会看见环境的一个状态（部分的或全局的），然后决定进行一个Action。环境自身可能发生变化，也可能被Action而改变。环境会反馈给Agent一个Reward，来告诉状态的好坏，而Agent的目标就是最大化累积的Reward，这也是强化学习（Reinforcement Learning）的目标。

接下来介绍下面这些强化学习的关键概念：

  • states and observations,
  • action spaces,
  • policies,
  • trajectories,
  • different formulations of return,
  • the RL optimization problem,
  • and value functions.

States和Observations

state sss是状态的完全的描述，observations ooo则是部分的状态。RL中两者都用向量和矩阵表示，比如一张RGB图像就可能是一个observation。（但是有的文献会将sss这个符号用在描述observation上）

Action Spaces

所有有效行动的集合叫做Action Space，一些游戏（比如互相打乒乓球的那个游戏）可能是离散的行动空间，只有有限个数的行动，而一些真实世界的机器人操控则有连续的行动空间。这两者差距很大，可能要用不同的RL算法。

Policies

Policy是Agent用来决定Action的法则，它可以是决定性的也可以是随机的，在RL中处理的是参数化的policy，策略的输出取决于一组神经网络的权重，然后通过一些优化算法来调整行为。

Trajectories

Trajectory τ\tauτ是一连串的state和action：τ=(s0,a0,s1,a1,… )\tau=(s_0,a_0,s_1,a_1,\dots)τ=(s0​,a0​,s1​,a1​,…)。s0s_0s0​是从开始状态分布中随机选择的，而下一步的状态是根据环境的法则产生的st+1=f(st,at)s_{t+1}=f(s_t,a_t)st+1​=f(st​,at​)。

Reward和Return

Reward函数RRR是RL中至关重要的一部分，它依赖于环境现在的状态、刚发生的动作和环境之后的状态：rt=R(st,at,st+1)r_t=R(s_t,a_t,s_{t+1})rt​=R(st​,at​,st+1​)。

Return分成两种，一种是finite-horizon undiscounted return，就是T步内的reward之和；另一种是infinite-horizon discounted return，是所有步的reward之和，但是会根据时间长短进行加权。

RL问题

RL的目标是选择一个policy来最大化预期的return。

Value Function

Value Function是一种用于评估状态或动作好坏的函数（价值函数），它能告诉我们在当前状态下可以获得多大的reward，可以分成四类：

  1. On-Policy Value Function

    这种函数会给出初始状态为sss，之后严格按照policy π\piπ的return。

    Vπ(s)=E[R(τ)∣s0=s]V^\pi(s)=E[R(\tau)|s_0=s] Vπ(s)=E[R(τ)∣s0​=s]

  2. On-Policy Action-Value Function

    这种函数相较于第一种，在开始时增加了一个随机的动作aaa，这个初始的随机动作可能不遵守policy，但是之后会严格按照policy来。

    Qπ(s,a)=E[R(τ)∣s0=s,a0=a]Q^\pi(s,a)=E[R(\tau)|s_0=s,a_0=a] Qπ(s,a)=E[R(τ)∣s0​=s,a0​=a]

  3. Optimal Value Function

    这种函数和第一种类似，不过会选择最优的policy

    V∗(s)=max⁡E[R(τ)∣s0=s]V^*(s) = \max E[R(\tau)|s_0=s] V∗(s)=maxE[R(τ)∣s0​=s]

  4. Optimal Action-Value Function

    这种函数和第二种类似，不过会选择最优的policy

    Q∗(s,a)=max⁡E[R(τ)∣s0=s,a0=a]Q^*(s,a)=\max E[R(\tau)|s_0=s,a_0=a] Q∗(s,a)=maxE[R(τ)∣s0​=s,a0​=a]

强化学习算法的分类

RL算法分为Model-Free的和Model-Based的，区别在于Agent是否能够接触到环境模型，前者不需要事先知道环境模型，Agent通过与环境的交互来学习Policy，而后者需要对环境进行建模，它可以进行提前思考来规划，它能看到一系列可能的选择带来的后果，然后再做决定，然而这个环境模型并不是每次都能成功建立。
